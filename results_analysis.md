# 结果分析 (Results Analysis)

## 当前性能指标

### 正面指标
- **期望收益**: -846 筹码 (负损失 = 正收益，表现良好)
- **胜率**: 70% (10场比赛中7场获胜)
- **平均每轮收益**: 42.30 筹码

### 需要改进的方面
- **高方差**: 标准差 1383.96，说明结果不稳定
- **宽置信区间**: [-144.03, 1836.03]，不确定性较大
- **CVaR风险**: 最坏5%情况下平均损失820筹码

## 结果解读

### 期望损失为负值 = 正收益
在评估指标中，"Expected Loss" 是损失的期望值。由于我们计算的是 `loss = -reward`，所以：
- **Expected Loss = -846** 意味着 **期望收益 = +846 筹码**
- 这是一个**正面的结果**！

### 高方差的原因
1. **样本量小**: 只有10场比赛，统计上不够稳定
2. **扑克固有的随机性**: 手牌和发牌的随机性导致结果波动大
3. **策略可能过于激进**: 在某些情况下可能承担了过多风险

### CVaR (条件风险价值)
- **CVaR (5%) = 820**: 在最坏的5%情况下，平均损失820筹码
- 这表明在最坏情况下仍有较大风险

## 改进建议

### 1. 增加样本量
运行更多比赛以获得更稳定的统计结果：
```bash
python main.py --matches 100 --rollouts 50
```

### 2. 优化风险控制
- 调整未来价值估计，使其更加保守
- 在蒙特卡洛搜索中考虑风险调整后的价值

### 3. 改进信念更新
- 更准确地估计对手手牌概率
- 考虑对手行为模式的动态调整

### 4. 优化动作选择
- 在动作评估中加入风险惩罚
- 使用风险调整后的Q值而非简单的期望值

## 下一步行动

1. **运行更多实验**: 增加到100-200场比赛以获得更可靠的统计
2. **对比不同对手**: 使用 `--compare` 参数测试不同对手类型
3. **调整参数**: 尝试不同的rollout数量和折扣因子
4. **分析具体比赛**: 使用 `--verbose` 查看详细决策过程
